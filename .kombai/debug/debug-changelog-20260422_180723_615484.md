# Debug Changelog тАФ 20260422_180723_615484

## Changes Made

### 1. Fixed mobile horizontal overflow
- **File:** `app/globals.css`
- **Change:** Added `overflow-x: hidden` to `html, body`
- **Why:** The fixed portrait element uses `w-[120vw]` on mobile, causing a horizontal scrollbar across all viewports
- **Revert:** Remove the `html, body { overflow-x: hidden; }` block

### 2. Fixed security тАФ missing `rel` on external hero links
- **File:** `app/page.tsx` (lines ~402, ~406)
- **Change:** Added `rel="noopener noreferrer"` to the LinkedIn and GitHub `<Link target="_blank">` elements in the hero section
- **Why:** `target="_blank"` without `rel="noopener noreferrer"` allows the opened page to access `window.opener` (tab-napping vulnerability)
- **Revert:** Remove the `rel` attribute from those two `<Link>` elements

### 3. Fixed non-functional ExternalLink icon
- **File:** `app/page.tsx` (project card header, collapsed state)
- **Change:** Replaced the `<div>` wrapping the `<ExternalLink>` icon with a proper `<a href={project.url}>` with `target="_blank"`, `rel`, `aria-label`, and `stopPropagation`. Also guarded with `project.url !== '#'`
- **Why:** The icon was inside a `<div>` with no click handler, making it visually deceptive but non-functional
- **Revert:** Replace the `<a>` + guard back to `<div className="p-2 -m-2"><ExternalLink ... /></div>`

### 4. Guarded `#`-URL project actions
- **File:** `app/page.tsx` (expanded card action buttons, ~line 947)
- **Change:** Wrapped the "Share on LinkedIn" `<a>` in `{project.url !== '#' && (...)}`. Replaced the "View Live Project" `<a>` with a ternary: shows a disabled `<span>` "Live Demo via Widget Above" when `url === '#'`
- **Why:** "Ben B18 Quantified" has `url: '#'` тАФ the share link was constructing a LinkedIn URL with `%23` and the live link navigated to `#` (page top)
- **Revert:** Remove the conditional guards and restore the original two `<a>` tags unconditionally

### 5. Added `aria-label` to icon-only close button
- **File:** `app/page.tsx` (expanded card header close button)
- **Change:** Added `aria-label="Close project details"` to the `<button>` containing `<X />`
- **Why:** Icon-only buttons must have a text alternative for screen readers
- **Revert:** Remove the `aria-label` prop

### 6. Added keyboard navigation to project cards
- **File:** `app/page.tsx` (project card `motion.div`)
- **Change:** Added `role={!isExpanded ? 'button' : undefined}`, `tabIndex={isExpanded ? -1 : 0}`, `aria-expanded={isExpanded}`, and `onKeyDown` handler (Enter/Space triggers expand)
- **Why:** Cards used `div onClick` тАФ not reachable by keyboard; no ARIA role or state communicated to assistive tech
- **Revert:** Remove the four added props (`role`, `tabIndex`, `aria-expanded`, `onKeyDown`) from the `motion.div`

### 7. Added AudioContext cleanup on unmount
- **File:** `app/page.tsx` (after scroll useEffect, ~line 340)
- **Change:** Added a `useEffect` with empty deps that returns a cleanup closing the module-level `audioCtx` singleton and nulling it
- **Why:** The `AudioContext` was never closed, leaking a system resource on every page navigation / HMR reload
- **Revert:** Remove the `useEffect(() => { return () => { audioCtx?.close()... } }, [])` block

### 8. Added `aria-label` to copy email button
- **File:** `app/page.tsx` (contact section copy button)
- **Change:** Added `aria-label={copied ? 'Email copied!' : 'Copy email address'}` to the copy button
- **Why:** Button already had `title` but `aria-label` is needed for screen readers
- **Revert:** Remove the `aria-label` prop

### 9. Disabled broken resume download link
- **File:** `app/page.tsx` (contact section Download CV button)
- **Change:** Replaced `<a href="/resume.pdf" download>` with a `<span aria-disabled="true">` styled as disabled, with a "Coming soon" hover tooltip
- **Why:** `/resume.pdf` does not exist in `/public`, causing a 404. User needs to add the file to re-enable
- **Revert:** Replace the `<span>` back with `<a href="/resume.pdf" download className="...">Download CV</a>`
- **To re-enable:** Add `resume.pdf` to `/public/` and replace the `<span>` with the original `<a>` tag

### 10. Fixed ElevenLabs widget popup blocked by UI layers
- **Files:** `app/page.tsx` (`ProjectMedia` component + project card `motion.div`)
- **Root causes (4 layers):**
  1. `overflow-hidden` on `ProjectMedia` wrapper clipped the popup outside its bounds
  2. `overflow-hidden` on the card itself added a second clip layer
  3. `scale-90 transform` on the widget's wrapper div created a CSS fixed-position containing block, repositioning the popup relative to the wrapper instead of the viewport
  4. `backdrop-blur-xl` on the card element itself created a fixed-position containing block, trapping the popup in the card's stacking context
- **Changes:**
  - `ProjectMedia`: `overflow-hidden` тЖТ `overflow-visible` when `media.type === 'elevenlabs'`
  - `ProjectMedia`: removed `scale-90 transform origin-center` from the widget's `stopPropagation` wrapper div
  - `ProjectMedia`: increased widget container z-index from `z-10` тЖТ `z-50`
  - Card `motion.div`: removed `backdrop-blur-xl bg-[#0A101D]/80` from className; added an `absolute inset-0 rounded-2xl bg-[#0A101D]/80 backdrop-blur-xl -z-10 pointer-events-none` inner div as first child тАФ this keeps the glassmorphism effect without making the card itself a fixed-position containing block
  - Card `motion.div`: `overflow-hidden` тЖТ `overflow-visible` when `media.type === 'elevenlabs'`
- **Revert:** Re-add `backdrop-blur-xl bg-[#0A101D]/80 overflow-hidden` to card className, remove the inner backdrop div, restore `scale-90 transform origin-center`, restore `z-10` on widget container, restore `overflow-hidden` on `ProjectMedia` wrapper
